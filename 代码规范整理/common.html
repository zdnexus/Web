基本原则

  1. 结构(HTML)、样式(CSS)、行为(JavaScript)分离

    尽量确保文档和模板只包含 HTML 结构，样式都放到 CSS 样式表里，行为都放到 JavaScript 脚本里

  2. 缩进

    统一(四个空格)缩进，不要使用 Tab 或者 Tab、空格混搭，因为windows下的tab键是占四个空格的位置，而在linux下会变成占八个空格的位置

  3. 文件编码

    • 在 HTML 中指定编码 <meta charset="utf-8">

    • 在 CSS 中无需使用 @charset 指定样式表的编码，它默认为 UTF-8

    • 在 JavaScript 中最好加上'user strict'严格模式，为ES6做准备(待商议)

    • HTML CSS JavaScript中每个样式/属性/函数/方法后加 ";" 方便压缩工具"断句"

  4. 一律使用小写字母

    
<!-- Good -->
<img src="pingan.png" alt="pingan"/>

<!-- Not Good -->
<A HREF="/">Home</A>
                    
    
<!-- Good -->
color: #e5e5e5;

<!-- Not Good -->
color: #E5E5E5;

  5. 省略外链资源 URL 协议部分

    省略外链资源（图片及其它媒体资源）URL 中的 http / https 协议，使 URL 成为相对地址，避免 Mixed Content 问题，减小文件字节数。其它协议（ftp 等）的 URL 不省略。

    
<!-- Good -->
<script src="//www.pingan.com/index.js"></script>

<!-- Not Good -->
<script src="http://www.pingan.com/index.js"></script>
                    
    
<!-- Good -->
.example {
    background: url(//www.pingan.com/index.png);
}

<!-- Not Good -->
.example {
    background: url(http://www.pingan.com/index.png);
}

  6. 注释

    • 文件注释

      文件注释用于告诉不熟悉这段代码的读者这个文件中包含哪些东西。 应该提供文件的大体内容, 它的作者, 依赖关系和兼容性信息。如下:

      
/**
 * @fileoverview Description of file, its uses and information
 * about its dependencies.
 * @author user@pingan.com (Firstname Lastname)
 * Copyright 2016 PingAn Inc. All Rights Reserved.
 */
    • HTML注释
      • 模块注释
        
<!-- 文章列表模块 -->
<div class="article-list">
...
</div>
      • 区域注释
        
<!--
@name: Drop Down Menu
@description: Style of top bar drop down menu.
@author: user(user@pingan.com )
-->
    • CSS注释
      • 模块注释
        
/* 文章列表模块 */
.article-list {
    color: #000;
}
    • JavaScript注释
      • 单行注释

        必须独占一行。// 后跟缩进与下一行被注释说明的代码一致。

      • 多行注释

        避免使用 /*...*/ 这样的多行注释。有多行注释内容时，使用多个单行注释。

      • 函数/方法注释

        函数/方法注释必须包含函数说明，有行参和返回值时必须使用注释标识(对象类型，对象注释，说明)。
