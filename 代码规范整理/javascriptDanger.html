JavaScript陷阱

  1. 申请变量必须以var来申请，没有用var申请的变量是全局变量会变成不可控因素

    
function func(){
  b = 0;
}
func();
console.log(b);//0 b为全局变量
                
    
function func(){
  var a = b = 0;
}
func();
console.log(b);//0 b为全局变量

  2. object对象赋值的时候是引用而不是复制

    
var arr=[1,2,3];
var foo = arr;
foo[2]=10;
console.log(arr);//1,2,10
console.log(foo);//1,2,10
console.log(arr == foo);//true
console.log(arr === foo);//true

  3. input，button，select的value值typeof类型为string,作比较的时候用==即可

  4. ECMAScript认为undefined是从null派生出来的，所以把它们定义为相等(undefined==null//true)的。但是，在(undefined===null//false)(typeof undefined == typeof null//false)的情况下是不一样的 

    
console.log(undefined == null);//true
console.log(undefined === null);//false
console.log(typeof undefined === typeof null);//false

  5. 新建立没有赋值的变量,判断该变量用undefined而不是null

  6. document.getElementById()等函数如果没有取得dom元素返回的是null而不是undefined

    
console.log(document.getElementById('demo'));//null

  7. if(x)里如果x = ''，x = null，x = undefined都不满足该条件

    
var a = '',b = null,c = undefined;
if(a){
  console.log('true');
}
else{
  console.log('false');//false
}

if(b){
  console.log('true');
}
else{
  console.log('false');//false
}

if(c){
  console.log('true');
}
else{
  console.log('false');//false
}
