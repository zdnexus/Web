CSS规范

  1. 为了防止页面css相互污染，在style后面加一个scoped属性表示此css为页面私有

    
<!-- Good -->
<style scoped="scoped">
</style>

  2. 公共属性可写在一个css里以供加载

  3. 写没有父子级CSS元素的情况下,最好不要直接用tagName,用className代替.会污染整个页面的tagName

    
<input type="text" class="example">

<!-- Good -->
.example {
    with:100px;
}

<!-- Not Good -->
input[type='text'] {
    with:100px;
}

  4. 0后面不跟单位(如0px)，写0即可

  5. 尺寸前面的数值0省掉(0.5em)，写.5em即可

  6. 颜色遇到相同字母时可缩减，比如#000000可写成#000,#AABBCC可写成#ABC

  7. url()中地址不加引号

  8. 涉及到有margin属性请写在一起,(padding同理,background,font也一样),方便代码压缩

    
<!-- Good -->
.example {
    margin:10px 10px 0 0;
}

<!-- Not Good -->
.example {
    margin-top:10px;
    margin-right:10px;
}

  9. 元素 id 必须保证页面唯一。

  10. className编写时采用前缀加中划线的形式，单词全字母小写，单词间以 - 分隔，描述清楚的前提下尽可能短

    导航页面的按钮
    
<!-- Good -->
#nav .nav-btn-group .nav-btn-sub{
}

  11. class 必须代表相应模块或部件的内容或功能，不得以样式信息进行命名。

    
<!-- Good -->
<div class="sidebar"></div>

<!-- Not Good -->
<div class="left"></div>

  12. 选择器尽可能简单,如#exa而不是div#exa

  13. 属性的顺序按可照个人喜好编写，可供参考:显示类型，尺寸，位置定位，字体,字体大小，字体颜色和背景颜色，内填充，外边距，边框

  14. div的默认宽度是父级宽度的100%，推荐还是写入比较好

  15. 每个声明最后都要有分号（即使是最后一个）

  16. 使用浮动的时候注意BFC清除浮动(1.父级元素使用伪元素前后添加内容2.父级元素本身添加overflow或者float属性)

  17. 在有规则的子元素顺序下优先选择first-of-type,弄清楚first-of-type和first-child的区别(其他同理)

  18. 在 CSS 可以实现相同需求的情况下不得使用表格进行布局
